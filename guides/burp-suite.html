What is Burp Suite?

Burp Suite is an integrated platform for performing security testing of web applications. Its various tools work seamlessly together to support the entire testing process, from initial mapping and analysis of an application's attack surface, through to finding and exploiting security vulnerabilities. It acts as a proxy, sitting between your browser and the target application, allowing you to intercept, inspect, and modify the raw traffic passing in both directions.

Initial Setup: Proxy Configuration

The most crucial first step is configuring your browser to work with Burp Suite.

Step 1: Configure Burp Proxy Listener

In Burp, go to the Proxy tab, then the Options sub-tab. Ensure the proxy listener is active and running on a specific address and port, typically 127.0.0.1:8080.

Step 2: Configure Your Browser

Set your browser's network settings to use a manual proxy configuration, pointing it to the same address and port as the Burp listener (127.0.0.1:8080). For HTTPS traffic, you must also install Burp's CA certificate in your browser.

# In your browser, visit http://burpsuite
# Click on 'CA Certificate' to download it, then import it into your browser's certificate store.

Step 3: Turn on Intercept

In Burp's Proxy tab, go to the Intercept sub-tab and click the "Intercept is off" button to toggle it on. Your browser traffic will now be paused in Burp for you to inspect and manipulate.

Core Tools & Usage

Here are some of the most important tools within Burp Suite.

Target

This tool gives you a site map of the target application, showing all the pages and resources you've discovered. It's the central hub for understanding the application's structure.

Proxy

The heart of Burp. It allows you to intercept, view, and modify all requests and responses between your browser and the target server. Use the Forward button to send the request on, Drop to discard it, or modify it before forwarding.

Intruder

A powerful tool for automating customized attacks. You can send a request to Intruder, mark positions in the request where you want to insert payloads, and configure payload lists (e.g., lists of common passwords, usernames, or fuzzing strings) to test for vulnerabilities like SQL injection or brute-force attacks.

Repeater

A simple but invaluable tool. Send any request to Repeater to manually modify and resend it over and over, observing how the response changes. This is perfect for manually testing for vulnerabilities or confirming findings from other tools.

Sequencer

Used to analyze the randomness of session tokens or other important data items that are intended to be unpredictable. It can help identify weak session management.

Tips for Beginners

  • Use the Site Map: Always start by exploring the application manually with the proxy enabled to build out a comprehensive site map in the Target tab.
  • Send to Repeater: Right-click on almost any request in Burp and choose "Send to Repeater." This is the fastest way to start manually testing an endpoint.
  • Don't Forget to Turn Intercept Off: It's easy to leave intercept on and wonder why your browser is hanging. Remember to toggle it off when you're just browsing the application.
  • Community Edition is Powerful: While the Professional version has more automation, the free Community Edition is extremely powerful and sufficient for most manual testing tasks.
