What is dirsearch?

dirsearch is a simple command-line tool designed to brute force directories and files in websites. It's written in Python and offers a fast, flexible approach to web content discovery. Unlike some other directory brute-forcers, dirsearch is designed to be simple to use while still offering advanced features like custom wordlists, multiple extensions, request customization, and result filtering. It's particularly popular among penetration testers and bug bounty hunters for its ease of use and effectiveness in discovering hidden web content.

Installation & Setup

Linux Installation

# Clone from GitHub
git clone https://github.com/maurosoria/dirsearch.git
cd dirsearch

# Install dependencies
pip3 install -r requirements.txt

# Make executable
chmod +x dirsearch.py

# Create symlink for global access
sudo ln -s $(pwd)/dirsearch.py /usr/local/bin/dirsearch

# Verify installation
python3 dirsearch.py -h

Automation and Scripting

Batch Scanning

# Multi-target scanning script
#!/bin/bash
targets=("http://example1.com" "http://example2.com" "http://example3.com")
output_dir="dirsearch_results"
mkdir -p "$output_dir"

for target in "${targets[@]}"; do
    echo "Scanning $target"
    domain=$(echo "$target" | sed 's|https\?://||' | sed 's|/.*||')
    python3 dirsearch.py -u "$target" -e php,html,js,txt \
                        -o "$output_dir/${domain}_results.txt" \
                        --format simple
    sleep 5
done

Result Processing

# Extract URLs from results
grep -E "^200|^301|^302" results.txt | awk '{print $3}' > found_urls.txt

# Parse JSON results
jq -r '.results[] | select(.status == 200) | .url' results.json > status_200.txt

# Python script for processing
import json

with open('results.json', 'r') as f:
    data = json.load(f)

interesting_files = []
for result in data['results']:
    if result['status'] in [200, 301, 302]:
        if any(ext in result['path'] for ext in ['.php', '.asp', '.jsp']):
            interesting_files.append(result['url'])

print(f"Found {len(interesting_files)} interesting files")

Pro Tips & Best Practices

  • Start with common wordlists: Use well-known lists before custom ones
  • Use appropriate extensions: Target relevant file types for the technology stack
  • Filter noise: Exclude common error pages and irrelevant responses
  • Tune performance: Adjust threads and delays based on target response
  • Use recursion carefully: Recursive scanning can generate many requests
  • Save results: Always output results to files for later analysis
  • Validate findings: Manually verify interesting discoveries
  • Combine techniques: Use with other reconnaissance tools
  • Respect rate limits: Avoid overwhelming target servers
  • Document discoveries: Keep organized records of findings
