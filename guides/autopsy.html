What is Autopsy?

Autopsy is a digital forensics platform and graphical interface to The Sleuth Kit and other digital forensics tools. It is used by law enforcement, military, and corporate examiners to investigate what happened on a computer. You can even use it to recover photos from your camera's memory card. Autopsy provides a comprehensive suite of tools for analyzing disk images, file systems, and digital evidence, making it an essential tool for digital forensics professionals and incident response teams.

Installation & Setup

Windows Installation

# Download from official website
# Visit: https://www.autopsy.com/download/

# Download the Windows installer
# Run the .msi installer as Administrator
# Follow the installation wizard

# System Requirements:
# - Windows 7 or later (64-bit recommended)
# - Java 8 or later
# - Minimum 4GB RAM (8GB+ recommended)
# - 1GB free disk space for installation

Linux Installation

# Ubuntu/Debian installation
sudo apt update
sudo apt install autopsy

# Install dependencies
sudo apt install sleuthkit testdisk

# Alternative: Install from source
wget https://github.com/sleuthkit/autopsy/releases/download/autopsy-4.20.0/autopsy-4.20.0.zip
unzip autopsy-4.20.0.zip
cd autopsy-4.20.0

# Install Java and dependencies
sudo apt install openjdk-8-jdk ant

# Build Autopsy
ant

macOS Installation

# Install Java first
brew install openjdk@8

# Download Autopsy for macOS
# Visit: https://www.autopsy.com/download/

# Extract and run
unzip autopsy-4.20.0.zip
cd autopsy-4.20.0/bin
./autopsy

Verification

# Verify installation (Linux)
autopsy --version

# Check Sleuth Kit installation
tsk_version

# Test basic functionality
mmls --help
fls --help

Getting Started

Starting Autopsy

# Windows: Use desktop shortcut or Start menu

# Linux: Command line
autopsy

# macOS: From application folder or terminal
./autopsy

# Web interface (if using older versions)
# Open browser to: http://localhost:9999/autopsy

Creating a New Case

  1. Launch Autopsy
  2. Click "New Case"
  3. Enter case information:
    • Case Name
    • Case Number
    • Examiner Name
    • Case Directory
  4. Click "Next" and "Finish"

Adding Data Sources

  1. Click "Add Data Source"
  2. Select data source type:
    • Disk Image or VM File
    • Local Drive
    • Logical Files
    • Unallocated Space Image File
  3. Browse and select your evidence
  4. Configure ingest modules
  5. Start analysis

Interface Overview

Main Interface Components

  • Tree Viewer: Hierarchical view of data sources and results
  • Result Viewer: Displays analysis results and file listings
  • Content Viewer: Shows file content, metadata, and hex dumps
  • Keyword Search: Search functionality across all data
  • Timeline: Chronological view of file system events
  • Communications: Email, chat, and call logs
  • Images/Videos: Media file analysis and viewing

Navigation Tips

  • Use the tree viewer to navigate file system structure
  • Right-click for context menus and additional options
  • Use tabs to keep multiple views open
  • Bookmark important findings for easy reference
  • Use filters to narrow down large result sets

Ingest Modules

Core Ingest Modules

  • Recent Activity: Web browsing, downloads, searches
  • Hash Lookup: Compare files against known hash databases
  • File Type Identification: Identify files by content, not extension
  • Keyword Search: Search for specific terms across all data
  • Email Parser: Extract and analyze email messages
  • Embedded File Extractor: Extract files from compound documents
  • EXIF Parser: Extract metadata from images
  • Extension Mismatch Detector: Find files with suspicious extensions

Advanced Modules

  • Photorec Carver: Recover deleted files
  • Plaso: Timeline analysis
  • Volatility: Memory analysis (if memory dumps available)
  • Android Analyzer: Android device analysis
  • iOS Analyzer: iOS device analysis
  • VM Extractor: Extract files from virtual machines

Legal Disclaimer

WARNING: Autopsy should only be used for:

  • Analyzing your own devices and data
  • Authorized forensic investigations with proper legal authority
  • Corporate investigations with appropriate authorization
  • Academic research with proper permissions
  • Educational purposes in controlled environments

Unauthorized access to computer systems or data may violate computer crime laws, privacy statutes, and other regulations. Digital forensics should only be performed by qualified professionals with proper legal authority. Always ensure compliance with applicable laws, regulations, and organizational policies. Maintain proper chain of custody and documentation for any evidence that may be used in legal proceedings. Be aware of privacy rights and data protection requirements in your jurisdiction.
