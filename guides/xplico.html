What is Xplico?

Xplico is an open-source Network Forensic Analysis Tool (NFAT) designed to extract and reconstruct application data from network traffic captures, such as PCAP files or live acquisitions. Unlike traditional protocol analyzers, Xplico focuses on reconstructing data for protocols like HTTP, SIP, IMAP, POP, SMTP, FTP, and VoIP, extracting artifacts such as emails, web content, and VoIP calls. It features a web-based user interface and supports SQLite, MySQL, or PostgreSQL backends, making it suitable for forensic investigators, incident responders, and cybersecurity professionals. Xplico is included in distributions like Kali Linux, DEFT, and Security Onion.

Installation & Setup

Ubuntu/Debian Installation

# Add Xplico repository
sudo bash -c 'echo "deb http://repo.xplico.org/ $(lsb_release -s -c) main" >> /etc/apt/sources.list'
sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys 791C25CE
sudo apt update
sudo apt install xplico

# Start Xplico service
sudo service xplico start

# Initialize SQLite database
sudo /opt/xplico/script/sqlite_demo.sh

# Verify installation
xplico -v

CentOS Installation

# Install dependencies
sudo yum groupinstall "Development Tools"
sudo yum install tcpdump httpd php php-mysql sqlite sqlite-devel libzip-devel recode sox php-gd php-xml php-mbstring php-ldap php-pear php-xmlrpc mysql-connector-odbc libdbi-dbd-mysql libnet libnet-devel libXext libXext-devel postgresql-devel psmisc python-pip
pip install psycopg2 httplib2

# Install MySQL
sudo rpm --ivh http://dev.mysql.com/get/mysql-community-release-el7-5.noarch.rpm
sudo yum install mysql-community-server mysql-community-client mysql-community-libs mysql-community-devel
sudo mysql_secure_installation
sudo chkconfig mysql on

# Download and compile Xplico
git clone https://github.com/xplico/xplico.git
cd xplico
make DISABLE_GEOIP=1 LOCAL_NDPI=1
sudo make install

# Configure MySQL database
cd /opt/xplico/script/db/mysql
./create_xplico_db.sh root xplico
cd /opt/xplico/cfg
mv dema.cfg dema_lite.cfg
cp dema_my.cfg dema.cfg
sed -i 's/DISPATCH=disp_lite.so/DISPATCH=disp_mysql.so/' *.cfg
sed -i 's/DB_TYPE=sqlite/DB_TYPE=mysql/' dema.cfg
sed -i 's/DB_NAME=xplico/DB_NAME=xplico/' dema.cfg
sed -i 's/DB_USER=xplico/DB_USER=xplico/' dema.cfg
sed -i 's/DB_PASSWORD=xplico/DB_PASSWORD=xplico/' dema.cfg

# Start Xplico
sudo /opt/xplico/script/mysql_demo.sh

Automation and Scripting

Bash Automation

#!/bin/bash
pcaps=("/pcap/capture1.pcap" "/pcap/capture2.pcap")
for pcap in "${pcaps[@]}"; do
    echo "Analyzing $pcap"
    xplico -m pcap -f "$pcap" -o "$(basename $pcap .pcap)_output"
done

Pro Tips & Best Practices

  • Use sandbox environments: Analyze PCAPs in isolated environments to avoid executing malicious content
  • Capture with Wireshark/tcpdump: Generate clean PCAPs for Xplico analysis
  • Enable GeoMap: Use KML output for geographic visualization
  • Filter protocols: Load only necessary dissectors to improve performance
  • Backup PCAPs: Preserve original files before analysis
  • Use MySQL for large cases: SQLite may be slow for large datasets
  • Monitor resources: Large PCAPs require significant RAM and CPU
  • Check for updates: Xplico is under active development, report bugs to bug@xplico.org
  • Combine tools: Use with Wireshark or NetworkMiner for comprehensive analysis
  • Document findings: Save logs, XML metadata, and exported files for reporting
