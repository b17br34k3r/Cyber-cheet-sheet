What is Nikto?

Nikto is an Open Source web server scanner which performs comprehensive tests against web servers for multiple items, including over 6700 potentially dangerous files/programs, checks for outdated versions of over 1250 servers, and version specific problems on over 270 servers. It also checks for server configuration items such as the presence of multiple index files, HTTP server options, and will attempt to identify installed web servers and software. Nikto is built on LibWhisker2 and can run on any platform which has a Perl environment.

Simple Scans

Automation and Integration

Batch Scanning

# Scan multiple targets from file
cat targets.txt
http://example1.com
http://example2.com
https://example3.com

nikto -h targets.txt -o batch_results.html -Format html

# Automated scanning script
#!/bin/bash
for target in $(cat targets.txt); do
    echo "Scanning $target"
    nikto -h "$target" -o "nikto_$(echo $target | sed 's/[^a-zA-Z0-9]/_/g').txt"
    sleep 5
done

CI/CD Integration

# Jenkins pipeline
pipeline {
    agent any
    stages {
        stage('Security Scan') {
            steps {
                sh 'nikto -h ${TARGET_URL} -o nikto-report.xml -Format xml'
                publishHTML([
                    allowMissing: false,
                    alwaysLinkToLastBuild: true,
                    keepAll: true,
                    reportDir: '.',
                    reportFiles: 'nikto-report.html',
                    reportName: 'Nikto Security Report'
                ])
            }
        }
    }
}

Legal Disclaimer

WARNING: Nikto should only be used for:

  • Testing web servers you own or have explicit permission to test
  • Authorized penetration testing with proper legal agreements
  • Security assessments with documented scope and approval
  • Educational purposes in controlled lab environments
  • Compliance scanning with appropriate authorization

Unauthorized web server scanning may violate computer crime laws, terms of service, and other regulations. Web server security testing should only be performed by qualified professionals with proper authorization and legal agreements. Nikto can generate significant traffic and may impact server performance or trigger security controls. Always ensure you have explicit written permission before scanning any web server. Some scan techniques may be considered intrusive and could potentially cause service disruption. Maintain proper documentation of testing activities and findings. Follow responsible disclosure practices for any vulnerabilities discovered. Be aware that scanning activities may be logged and monitored by target systems. Respect rate limits and implement appropriate delays to minimize impact on target systems.
