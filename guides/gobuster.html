What is Gobuster?

Gobuster is a tool used to brute-force URIs (directories and files) in web sites, DNS subdomains (with wildcard support), Virtual Host names on target web servers, and Amazon S3 buckets. Written in Go, Gobuster is fast, efficient, and supports multiple modes of operation. It's particularly useful for web application security testing, penetration testing, and bug bounty hunting to discover hidden directories, files, and subdomains that might contain sensitive information or additional attack surfaces.

Basic Usage

Directory Brute-Forcing

# Basic directory scan
gobuster dir -u http://example.com -w /usr/share/wordlists/dirb/common.txt

# Scan with specific extensions
gobuster dir -u http://example.com -w wordlist.txt -x php,html,txt,js

# Scan with custom user agent
gobuster dir -u http://example.com -w wordlist.txt -a "Custom User Agent"

# Scan with authentication
gobuster dir -u http://example.com -w wordlist.txt -U username -P password

DNS Subdomain Enumeration

# Basic subdomain scan
gobuster dns -d example.com -w /usr/share/wordlists/dnsrecon/subdomains-top1mil-5000.txt

# Scan with custom resolvers
gobuster dns -d example.com -w subdomains.txt -r 8.8.8.8,1.1.1.1

# Wildcard detection
gobuster dns -d example.com -w subdomains.txt --wildcard

# Show IP addresses
gobuster dns -d example.com -w subdomains.txt -i

Virtual Host Discovery

# Virtual host enumeration
gobuster vhost -u http://example.com -w vhosts.txt

# Append domain to wordlist
gobuster vhost -u http://example.com -w subdomains.txt --append-domain

# Custom headers
gobuster vhost -u http://example.com -w vhosts.txt -H "Host: {GOBUSTER}.example.com"

Advanced Options

Performance Tuning

# Increase threads for faster scanning
gobuster dir -u http://example.com -w wordlist.txt -t 50

# Add delay between requests
gobuster dir -u http://example.com -w wordlist.txt --delay 100ms

# Set timeout
gobuster dir -u http://example.com -w wordlist.txt --timeout 10s

# Limit request rate
gobuster dir -u http://example.com -w wordlist.txt -t 10 --delay 200ms

Response Filtering

# Hide specific status codes
gobuster dir -u http://example.com -w wordlist.txt -b 404,403

# Show only specific status codes
gobuster dir -u http://example.com -w wordlist.txt -s 200,301,302

# Hide responses by length
gobuster dir -u http://example.com -w wordlist.txt --exclude-length 1234

# Hide responses by content
gobuster dir -u http://example.com -w wordlist.txt --exclude-content "Not Found"

Output Options

# Save output to file
gobuster dir -u http://example.com -w wordlist.txt -o results.txt

# Verbose output
gobuster dir -u http://example.com -w wordlist.txt -v

# Quiet mode (minimal output)
gobuster dir -u http://example.com -w wordlist.txt -q

# No progress indicator
gobuster dir -u http://example.com -w wordlist.txt --no-progress

Automation and Scripting

Batch Scanning

# Scan multiple targets
#!/bin/bash
targets=("http://example1.com" "http://example2.com" "http://example3.com")
wordlist="/usr/share/wordlists/dirb/common.txt"

for target in "${targets[@]}"; do
    echo "Scanning $target"
    gobuster dir -u "$target" -w "$wordlist" -o "$(echo $target | sed 's|http://||' | sed 's|/|_|g').txt"
    sleep 5
done

Integration with Other Tools

# Combine with Nmap results
nmap -p 80,443,8080,8443 --open target_network | grep -E "^[0-9]" | awk '{print "http://"$1}' > targets.txt
while read target; do
    gobuster dir -u "$target" -w wordlist.txt -o "gobuster_$(echo $target | tr '/:' '_').txt"
done < targets.txt

# Parse results for further analysis
grep -h "Status: 200" gobuster_*.txt | awk '{print $1}' | sort -u > found_directories.txt

# Python script for processing results
import json
with open('results.json', 'r') as f:
    results = json.load(f)

    if result['status'] in [200, 301, 302]:

Pro Tips & Best Practices

  • Start with common wordlists: Use well-known lists before custom ones
  • Tune for target: Adjust threads and delays based on server response
  • Use multiple wordlists: Different lists may reveal different content
  • Check extensions: Include relevant file extensions for the technology stack
  • Monitor responses: Look for unusual status codes or response lengths
  • Combine techniques: Use directory and subdomain enumeration together
  • Validate findings: Manually verify interesting discoveries
  • Respect rate limits: Avoid overwhelming target servers
  • Document results: Keep organized records of discoveries
  • Follow up: Investigate discovered directories and files further

Legal Disclaimer

WARNING: Gobuster should only be used for:

  • Testing web applications you own or have explicit permission to test
  • Authorized penetration testing with proper legal agreements
  • Bug bounty programs with defined scope and rules
  • Educational purposes in controlled lab environments
  • Security research with appropriate institutional approval

Unauthorized directory brute-forcing and subdomain enumeration may violate computer crime laws, terms of service, and other regulations. Web application security testing should only be performed by qualified professionals with proper authorization and legal agreements. Gobuster can generate significant traffic and may impact server performance or trigger security controls. Always ensure you have explicit written permission before scanning any web application or domain. Some scanning techniques may be considered intrusive and could potentially cause service disruption. Maintain proper documentation of testing activities and findings. Follow responsible disclosure practices for any vulnerabilities or sensitive information discovered. Be aware that scanning activities may be logged and monitored by target systems. Respect rate limits and implement appropriate delays to minimize impact on target systems. DNS enumeration may reveal sensitive internal infrastructure information that should be handled appropriately.
