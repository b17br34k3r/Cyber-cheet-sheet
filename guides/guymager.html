What is Guymager?

Guymager is a free, open-source forensic imaging tool for Linux, designed to create bit-by-bit copies (forensic images) of storage devices such as hard drives, USB drives, and memory cards. It features a Qt-based graphical user interface (GUI) for ease of use, supports multiple image formats (raw/dd, EWF/E01, AFF), and is optimized for speed with multi-threaded data compression and hashing (MD5, SHA-1, SHA-256). Guymager is pre-installed in forensic distributions like Kali Linux and CAINE, making it a popular choice for digital forensics and incident response to preserve evidence while maintaining data integrity.

Advanced Forensic Techniques

Enable AFF Format

# Edit configuration file to enable AFF
sudo nano /etc/guymager/local.cfg
# Add line:
AffEnabled=true

# Restart Guymager
sudo guymager

Custom Configuration

# Create custom config
guymager cfg=template.cfg

# Edit custom config
nano template.cfg
# Example: Set compression and threads
CompressionLevel=5
CompressionThreads=4

# Run with custom config
guymager cfg=template.cfg log=/var/log/custom_guymager.log

Verify Image Integrity

# Check .info file for hash values
cat /path/to/image.info

# Manually verify image hash
sha256sum /path/to/image.E01

# Use ewfverify for EWF files
ewfverify /path/to/image.E01

Automation and Scripting

Bash Automation

#!/bin/bash
# Script to automate imaging multiple devices
devices=("/dev/sdb" "/dev/sdc")
for device in "${devices[@]}"; do
    echo "Imaging $device"
    guymager cfg=/etc/guymager/guymager.cfg \
             log=/var/log/guymager_$(basename $device).log \
             --destination=/images/case001/$(basename $device).E01 \
             --case=CASE001 --evidence=$(basename $device) --examiner="John Doe"
done

Automate Directory Creation

#!/bin/bash
case="CASE001"
for i in {01..05}; do
    mkdir -p /images/$case/image$i
    guymager --destination=/images/$case/image$i/image$i.E01 \
             --case=$case --evidence=image$i --examiner="John Doe"
done

Specialized Forensic Techniques

Image Failing Drives

# Handle bad sectors
guymager --destination=/images/case001/failing.E01 --bad-sector-handling=skip

# Monitor progress
tail -f /var/log/guymager.log

Mount Images for Analysis

# Mount EWF image (read-only)
sudo ewfmount /images/case001/evidence001.E01 /mnt/ewf
ls /mnt/ewf

# Mount AFF image (if enabled)
sudo affuse /images/case001/evidence001.aff /mnt/aff
ls /mnt/aff

Resume Interrupted Imaging

# Guymager automatically resumes if same destination file is selected
sudo guymager
# Select same device and destination, Guymager will detect partial image

Troubleshooting

Common Issues

# Check device availability
lsblk

# Verify write-blocker status
sudo blockdev --getro /dev/sdc

# Debug Guymager
guymager log=/var/log/guymager_debug.log --verbose

# Check for missing dependencies
ldd /usr/bin/guymager

Performance Issues

# Monitor system resources
top

# Limit CPU usage
cpulimit -p $(pgrep guymager) -l 50

# Optimize compression
guymager cfg=/etc/guymager/guymager.cfg CompressionLevel=3

Bad Sector Handling

# Skip bad sectors
guymager --destination=/images/case001/image.E01 --bad-sector-handling=skip

# Log bad sectors
grep "bad sector" /var/log/guymager.log

  • Image compromised systems for malware analysis
  • Backup failing drives before data recovery
  • Acquire images from live systems (with caution)
  • Generate reports for incident documentation

Pro Tips & Best Practices

  • Use hardware write-blockers: Prevent accidental writes to source devices
    • [](https://notes.qazeer.io/dfir/common/image_acquisition_and_mounting)
  • Verify hashes: Always check source and image hashes to ensure integrity
    • [](https://www.cybrary.it/blog/creating-a-forensic-disk-image-using-the-linux-guymager-utility)
  • Enable EWF format: Use EWF for compression and metadata support
    • [](https://www.cqr.tools/tools/guymager)
  • Label evidence: Match physical labels with metadata for chain of custody
    • [](https://www.cybrary.it/blog/creating-a-forensic-disk-image-using-the-linux-guymager-utility)
  • Store images securely: Use encrypted drives for sensitive data
  • Monitor logs: Check /var/log/guymager.log for errors or bad sectors
  • Use forensic distributions: Boot CAINE or Kali Linux in forensic mode to prevent auto-mounting
    • [](https://notes.qazeer.io/dfir/common/image_acquisition_and_mounting)
  • Automate repetitive tasks: Script imaging for multiple devices
    • [](https://sourceforge.net/p/guymager/wiki/Home/)
  • Backup images: Store multiple copies on separate drives
    • [](https://sourceforge.net/p/guymager/wiki/Home/)
  • Test with small drives: Practice on USB drives before imaging large disks
