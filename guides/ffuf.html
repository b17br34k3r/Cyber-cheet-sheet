What is ffuf?

ffuf (Fuzz Faster U Fool) is a fast web fuzzer written in Go. It's designed to be simple, fast, and flexible for discovering hidden files, directories, parameters, and other web application components. ffuf excels at high-speed fuzzing with customizable wordlists, multiple fuzzing modes, and advanced filtering capabilities. It's particularly popular among bug bounty hunters and penetration testers for its speed and versatility in web application security testing.

Installation & Setup

Linux Installation

# Download from GitHub releases
wget https://github.com/ffuf/ffuf/releases/download/v2.1.0/ffuf_2.1.0_linux_amd64.tar.gz
tar -xzf ffuf_2.1.0_linux_amd64.tar.gz
sudo mv ffuf /usr/local/bin/

# Build from source
git clone https://github.com/ffuf/ffuf.git
cd ffuf
go build
sudo mv ffuf /usr/local/bin/

# Verify installation
ffuf -V

Basic Usage

Directory Fuzzing

# Basic directory fuzzing
ffuf -u http://example.com/FUZZ -w /usr/share/wordlists/dirb/common.txt

# Fuzz with file extensions
ffuf -u http://example.com/FUZZ -w wordlist.txt -e .php,.html,.txt,.js

# Recursive fuzzing
ffuf -u http://example.com/FUZZ -w wordlist.txt -recursion -recursion-depth 2

# Multiple wordlists
ffuf -u http://example.com/FUZZ -w wordlist1.txt:W1,wordlist2.txt:W2

Parameter Fuzzing

# GET parameter fuzzing
ffuf -u "http://example.com/search?FUZZ=test" -w parameters.txt

# POST parameter fuzzing
ffuf -u http://example.com/login -w parameters.txt -X POST -d "FUZZ=value"

# Multiple parameter fuzzing
ffuf -u "http://example.com/api?param1=FUZZ¶m2=W2" -w values1.txt:FUZZ,values2.txt:W2

# Header fuzzing
ffuf -u http://example.com -w headers.txt -H "FUZZ: value"

Subdomain Fuzzing

# Subdomain enumeration
ffuf -u http://FUZZ.example.com -w subdomains.txt

# With custom Host header
ffuf -u http://example.com -w subdomains.txt -H "Host: FUZZ.example.com"

# HTTPS subdomain fuzzing
ffuf -u https://FUZZ.example.com -w subdomains.txt

Advanced Fuzzing Techniques

Multiple Fuzzing Points

# Multiple FUZZ keywords
ffuf -u http://example.com/W1/W2 -w dirs.txt:W1,files.txt:W2

# Complex URL fuzzing
ffuf -u "http://FUZZ.example.com/W2?param=W3" -w subdomains.txt:FUZZ,paths.txt:W2,values.txt:W3

# Method and path fuzzing
ffuf -u http://example.com/FUZZ -w paths.txt -X W2 -w methods.txt:W2

Request Customization

# Custom headers
ffuf -u http://example.com/FUZZ -w wordlist.txt -H "Authorization: Bearer token" -H "User-Agent: ffuf"

# Custom cookies
ffuf -u http://example.com/FUZZ -w wordlist.txt -b "sessionid=abc123; token=xyz789"

# Custom data and content type
ffuf -u http://example.com/api -w payloads.txt -X POST -d "FUZZ" -H "Content-Type: application/json"

# Proxy usage
ffuf -u http://example.com/FUZZ -w wordlist.txt -x http://127.0.0.1:8080

Performance Tuning

# Increase threads
ffuf -u http://example.com/FUZZ -w wordlist.txt -t 100

# Add delays
ffuf -u http://example.com/FUZZ -w wordlist.txt -p 0.1-2.0

# Set timeout
ffuf -u http://example.com/FUZZ -w wordlist.txt -timeout 10

# Rate limiting
ffuf -u http://example.com/FUZZ -w wordlist.txt -rate 50

Response Filtering

Status Code Filtering

# Filter by status codes
ffuf -u http://example.com/FUZZ -w wordlist.txt -fc 404,403

# Match specific status codes
ffuf -u http://example.com/FUZZ -w wordlist.txt -mc 200,301,302

# Filter ranges
ffuf -u http://example.com/FUZZ -w wordlist.txt -fc 400-499

Size and Content Filtering

# Filter by response size
ffuf -u http://example.com/FUZZ -w wordlist.txt -fs 1234

# Match specific sizes
ffuf -u http://example.com/FUZZ -w wordlist.txt -ms 5000-10000

# Filter by word count
ffuf -u http://example.com/FUZZ -w wordlist.txt -fw 100

# Filter by line count
ffuf -u http://example.com/FUZZ -w wordlist.txt -fl 50

Content-Based Filtering

# Filter by regex pattern
ffuf -u http://example.com/FUZZ -w wordlist.txt -fr "error|not found"

# Match regex pattern
ffuf -u http://example.com/FUZZ -w wordlist.txt -mr "admin|login"

# Filter by response time
ffuf -u http://example.com/FUZZ -w wordlist.txt -ft 5000

Automation and Scripting

Bash Automation

# Multi-target fuzzing script
#!/bin/bash
targets=("http://example1.com" "http://example2.com")
wordlist="/path/to/wordlist.txt"

for target in "${targets[@]}"; do
    echo "Fuzzing $target"
    ffuf -u "$target/FUZZ" -w "$wordlist" -o "$(echo $target | sed 's|http://||' | sed 's|/|_|g').json" -of json
    sleep 2
done

Result Processing

# Extract URLs from JSON output
jq -r '.results[] | .url' results.json > found_urls.txt

# Filter by status code
jq -r '.results[] | select(.status == 200) | .url' results.json > status_200.txt

# Python script for processing
import json

with open('results.json', 'r') as f:
    data = json.load(f)

for result in data['results']:
    if result['status'] in [200, 301, 302]:
        print(f"Found: {result['url']} (Status: {result['status']}, Size: {result['length']})")

Integration with Other Tools

# Chain with other tools

jq -r '.results[] | .url' found.json | while read url; do

    curl -s "$url" | grep -i "admin\|login\|password"

Pro Tips & Best Practices

  • Start with small wordlists: Test with small lists before using large ones
  • Use appropriate filtering: Filter out noise to focus on interesting results
  • Tune performance: Adjust threads and delays based on target response
  • Save results: Always output results to files for later analysis
  • Use multiple fuzzing points: Combine different fuzzing techniques
  • Monitor target response: Watch for rate limiting or blocking
  • Validate findings: Manually verify interesting discoveries
  • Use recursion carefully: Recursive fuzzing can generate many requests
  • Customize for target: Adapt wordlists and techniques to specific applications
  • Combine with manual testing: Use ffuf results to guide manual exploration

Legal Disclaimer

WARNING: ffuf should only be used for:

  • Testing web applications you own or have explicit permission to test
  • Authorized penetration testing with proper legal agreements
  • Bug bounty programs with defined scope and rules
  • Educational purposes in controlled lab environments
  • Security research with appropriate institutional approval

Unauthorized web application fuzzing may violate computer crime laws, terms of service, and other regulations. Web application security testing should only be performed by qualified professionals with proper authorization and legal agreements. ffuf can generate significant traffic and may impact server performance, trigger security controls, or cause service disruption. Always ensure you have explicit written permission before fuzzing any web application. High-speed fuzzing can be particularly intrusive and may overwhelm target systems. Maintain proper documentation of testing activities and findings. Follow responsible disclosure practices for any vulnerabilities discovered. Be aware that fuzzing activities may be logged and monitored by target systems. Respect rate limits and implement appropriate delays to minimize impact on target systems. Some fuzzing techniques may trigger automated security responses or blocking mechanisms.
