What is Angry IP Scanner?

Angry IP Scanner (ipscan) is a very fast IP address and port scanner. It can scan IP addresses in any range as well as any their ports. It is cross-platform and lightweight, not requiring any installation. It simply pings each IP address to check if it's alive, then optionally it is resolving its hostname, determines the MAC address, scans ports, etc. The amount of gathered data about each host can be extended with plugins.

Command Line Usage

Command Line Options

# Basic command line scan
java -jar ipscan.jar -c -f:range 192.168.1.0/24

# Export to file
java -jar ipscan.jar -c -f:range 192.168.1.0/24 -o results.txt

# Specify output format
java -jar ipscan.jar -c -f:range 192.168.1.0/24 -f:csv -o results.csv

# Quiet mode
java -jar ipscan.jar -c -q -f:range 192.168.1.0/24

Batch Scanning Script

#!/bin/bash
# Batch network scanning script

networks=("192.168.1.0/24" "192.168.2.0/24" "10.0.0.0/24")
timestamp=$(date +%Y%m%d_%H%M%S)

for network in "${networks[@]}"; do
    echo "Scanning $network..."
    output_file="scan_${network//\//_}_${timestamp}.csv"
    java -jar ipscan.jar -c -q -f:range $network -f:csv -o $output_file
    echo "Results saved to: $output_file"
done

Troubleshooting & Common Issues

Java-Related Issues

# Check Java version
java -version

# Minimum Java 8 required
# Download from: https://www.oracle.com/java/technologies/downloads/

# Run with specific Java version

Network Issues

  • No results: Check network connectivity
  • Slow scanning: Reduce thread count or increase timeout
  • Firewall blocking: Configure firewall exceptions
  • Permission issues: Run as administrator if needed

Performance Issues

  • High memory usage: Reduce scan range or threads
  • Application freezing: Lower thread count
  • Network congestion: Add delays between requests
  • Incomplete results: Increase timeout values

Legal Disclaimer

WARNING: Angry IP Scanner should only be used for:

  • Scanning your own networks and systems
  • Authorized network administration and troubleshooting
  • Security assessments with proper written authorization
  • Educational purposes in controlled lab environments

Unauthorized network scanning may violate computer crime laws, network policies, and terms of service. Network scanning can be detected by intrusion detection systems and may be considered reconnaissance activity. Always obtain proper authorization before scanning networks you don't own or administer. Be respectful of network resources and avoid causing disruption through excessive scanning.
